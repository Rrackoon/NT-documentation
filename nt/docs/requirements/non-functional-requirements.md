# Нефункциональные требования

## Security

- Передача данных выполняется по HTTPS с TLS 1.2+
- Пароли хранятся в виде криптографического хеша, например `bcrypt` с salt и не менее 12 раундов
- Пользователь имеет доступ только к своим данным

## Performance

- Время отклика для 95-го перцентиля ключевых операций не более 2 секунд
- Целевая нагрузка для MVP - до 1 000 активных пользователей

## Reliability

- Доступность сервиса не ниже 99.9%
- Резервное копирование базы - не реже одного раза в сутки
- MTTR для некритичных инцидентов - до 30 минут

## Usability

- Добавление записи о приеме пищи должно занимать не более 5 действий
- Пользователь должен находить нужную функцию не более чем за 20 секунд

## Compatibility

- Android 10+
- iOS 14+
- Поддержка распространенных мобильных разрешений

## Scalability

- Архитектура должна выдерживать рост до 10 000 одновременно активных пользователей
- Масштабирование серверов и БД должно быть предусмотрено без полного перепроектирования

## Regulatory

- Обработка персональных данных должна соответствовать ФЗ-152
- Пользователь должен подтверждать согласие на обработку данных
- Должна существовать возможность удаления персональных данных по запросу

## Observability

- Логирование ошибок и бизнес-событий
- Мониторинг доступности и времени ответа
- Хранение логов не менее 30 дней
- Автоматическое уведомление при превышении порогов инцидентов
