# Event Storming

## Основной фокус

Текущий Event Storming концентрируется на сценарии добавления приема пищи и обновления дневного баланса

## Выделенные элементы

- событие: пользователь добавляет запись о приеме пищи
- команда: найти продукт, выбрать продукт, указать количество, сохранить запись
- агрегат: прием пищи
- read model: дневник питания, дневной баланс
- внешняя система: API базы продуктов
- hot spots: дублирование пользовательских блюд, ошибки пересчета, высокая нагрузка на поиск

## Выводы по модели

- Поиск продукта выделяется как отдельная важная зона нагрузки
- Создание пользовательского блюда должно быть встроено в основной сценарий без разрыва пользовательского потока
- Пересчет дневных итогов является обязательным пост-эффектом сохранения приема пищи
