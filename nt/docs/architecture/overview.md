# Архитектурный обзор

## Архитектурный стиль

NutriTrack проектируется как клиент-серверная система:

- мобильный клиент для iOS и Android
- backend API
- хранилища данных
- аналитический и рекомендательный контур для будущих релизов

## Основные подсистемы

- `Mobile App` - ввод данных, показ дневника, профиля и аналитики
- `Backend API` - бизнес-логика, авторизация, расчет КБЖУ, управление сущностями
- `PostgreSQL` - транзакционные сущности пользователя, приемов пищи и целей
- `TimescaleDB` - временные ряды по весу и аналитике
- `Elasticsearch` - поиск продуктов по названию
- `Redis` - кэш агрегированной статистики

## Ключевые архитектурные решения

- История питания хранится снапшотами через `MealItem`, чтобы изменение справочника продуктов не ломало прошлые записи
- Для поиска продуктов используется `PostgreSQL + Elasticsearch`
- Для временных рядов выбран `TimescaleDB`
- Для frequently-read статистики допускается кэширование через `Redis`
