# BPMN и decision table

## Основной процесс

Текущий BPMN-артефакт описывает поток добавления нового приема пищи:

1. Пользователь выбирает добавление записи
2. Система выполняет поиск продукта или пользователь создает собственное блюдо
3. Пользователь указывает количество
4. Сервис валидирует калорийность
5. Пользователь получает статус `OK`, `Warning` или `Recommendation`
6. После подтверждения запись сохраняется
7. Система пересчитывает дневной баланс и обновляет экран дневника

## Правила проверки калорийности

| Калории в текущем приеме | Суммарные калории | Цель пользователя | Результат | Уровень | Интерпретация |
| --- | --- | --- | --- | --- | --- |
| - | `> dailyGoal * 1.2` | - | Warning | High | Суточная норма значительно превышена |
| - | `> dailyGoal and <= dailyGoal * 1.2` | - | Warning | Medium | Есть превышение суточной нормы |
| - | `>= dailyGoal * 0.9 and <= dailyGoal` | - | Recommendation | Low | Пользователь близок к лимиту |
| - | `>= dailyGoal * 0.8 and < dailyGoal * 0.9` | - | Recommendation | Medium | Стоит уменьшить порцию |
| - | `< dailyGoal * 0.8` | - | OK | None | Все в порядке |
